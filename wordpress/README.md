# WordPress on containers

This is my attempt to create a local wordpress development environment based on containers.

## Inspiration

I was inspired by this YouTube video for the most part: https://www.youtube.com/watch?v=kIqWxjDj4IU

## Requirements

- docker/podman
- docker compose/podman-compose

## Included

- Caddy
- PHP
- MySQL
- WordPress

## Usage

1. Clone this repository and move into it.
2. Download the latest WordPress zip file from https://wordpress.org/download/ and unzip it in the `wordpress` folder.
3. Rename `wordpress/wp-config-sample.php` to `wordpress/wp-config.php`.
4. Update the `compose.yml` file and `wordpress/wp-config.php` file with your own settings (database name, user, password, etc).
5. After that you can run `docker compose up -d --build` to start the containers and `docker compose down` to stop them.
6. You can now access your WordPress site at `https://wp.local`.

> [!IMPORTANT]
> For SSL certs generated by this Caddy to be recognized by your browser, you have to trust the root certificated located at `wordpress/caddy_data/caddy/pki/authorities/local/root.crt`.

> [!TIP]
> Everywhere you see `wp.local` you can replace it with your own local domain.
